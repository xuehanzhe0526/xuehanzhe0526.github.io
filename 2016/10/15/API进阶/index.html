API进阶

Oct 15, 2016

摘要：

  • rate limiting
    -
    -

为了确保API能被所有用户获取和并且能相应用户的请求，API会限制你在短时间内进行大量的请求、这就是速率限制，确保了一个用户不会因为太多太快的请求而使API服务器超载。rate limiting

通常情况下想要通过API获取一个网站服务器的数据是需要授权的

本文会学习如何利用Github API从用户以及仓库中拉取一些有意思的数据

Github API 文档:
https://developer.github.com/v3/

Github Authentication:
https://developer.github.com/v3/#authentication

API Authentication(认证)

为了使得Github API得到Github的授权，我们需要利用访问令牌（access token）[https://github.com/settings/tokens]

一个访问令牌是一个凭证（是一个字符串）,有了这个凭证，API就可以读取以及关联你的账户。

用户Vik Paruchuri生成的令牌：1f36137fbbe1602f779300dad26e4c1b7fbab631
(This token is generated by Github, and is associated with the account of Vik Paruchuri)

Authorization header

{"Authorization": "token 1f36137fbbe1602f779300dad26e4c1b7fbab631"}
